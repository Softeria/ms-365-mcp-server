# Microsoft 365 MCP Catalog Navigation Guide

Use this guide when a discovery-mode tenant needs to navigate Microsoft 365 without expanding `tools/list` beyond the meta-tool surface.

## Discovery Loop

1. Use `search-tools` with a plain-language goal and a workload hint.
2. Use `get-tool-schema` on the best alias to inspect required parameters.
3. Use `execute-tool` only after the schema and permission fit are clear.

If the same alias is useful more than once, save it as a bookmark. If the same parameter shape repeats, save it as a recipe so later sessions can run it with only the changing values.

## Generated Resources To Read Next

`m365://catalog/scope-map.json` is the generated alias-to-scope map. Read it when choosing between similar aliases or when explaining why consent is needed.

`m365://endpoint/{alias}.schema.json` is generated when read by the resource dispatcher. Read it after `search-tools` finds a candidate and before `execute-tool` sends parameters.

Tenant-scoped resources, such as enabled tools, granted scopes, audit history, bookmarks, recipes, and facts, are also generated by the resource dispatcher. They are not part of this static markdown catalog.
